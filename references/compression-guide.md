# 压缩参数速查

## 视频压缩

| 格式 | CPU编码器 | GPU编码器(NVIDIA) | GPU编码器(AMD) | GPU编码器(Intel) |
|------|-----------|-------------------|---------------|-----------------|
| MP4  | libx264   | h264_nvenc        | h264_amf      | h264_qsv        |
| MKV  | libx265   | hevc_nvenc        | hevc_amf      | hevc_qsv        |
| WebM | libvpx-vp9| -                 | -             | -               |

## CRF值说明

| CRF | 质量 | 适用场景 |
|-----|------|----------|
| 18  | 最高 | 存档、专业用途 |
| 23  | 高 | 日常使用（默认） |
| 28  | 良好 | 网络传输、移动设备 |

## CPU vs GPU 选择建议

| 场景 | 推荐 | 原因 |
|------|------|------|
| 追求质量 | CPU (libx264) | CPU编码画质更优,同等码率更清晰 |
| 追求速度 | GPU (NVENC/QSV) | 快3-5倍,适合批量处理 |
| 日常使用 | CPU (libx264) | 兼容性最好,几乎所有设备可播放 |
| 4K视频 | GPU (HEVC) | 4K编码计算量大,GPU加速明显 |

## 常见问题

**Q: 压缩后画质有损失吗？**
A: 使用"平衡模式"(CRF 23)几乎看不出画质损失,使用"保持画质"(CRF 28)完全看不出。

**Q: GPU编码画质比CPU差吗？**
A: 同等码率下CPU编码画质略好(约10-15%),但GPU编码速度快3-5倍。日常使用看不出区别。

**Q: 支持哪些GPU？**
A: NVIDIA 600系列以上(NVENC)、AMD RX 5000系列以上(AMF)、Intel 7代以上(QSV)、Apple M系列(VideoToolbox)
